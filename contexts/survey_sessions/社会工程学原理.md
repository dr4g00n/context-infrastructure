# 社会工程学原理

## 核心定义

社会工程学（Social Engineering）是通过操纵人类心理弱点，获取敏感信息、访问权限或执行特定行动的"黑客艺术"。

> "最脆弱的防火墙是人的大脑"

---

## 为什么社会工程学有效

### 技术防御 vs 人类弱点

```
技术防御                 人类弱点
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
防火墙可以升级          人性无法修补
入侵检测可自动化        信任难以量化
漏洞可以扫描            认知偏差普遍存在
密码可以加密            社会工程无法加密
```

### 核心假设

**传统安全假设：**
- 攻击者会突破技术防御
- 需要加固系统边界

**社会工程学现实：**
- 攻击者会绕过技术防御
- 最短的路径是通过"人"

---

## 社会工程学的三大支柱

### 1. 信任建立
```
陌生 → 建立信任 → 利用信任 → 目标达成
  ↑                           ↓
  └─────────── 撤退/消失 ───────┘
```

**信任建立方法：**
- 权威伪装：伪造身份、职位
- 相似性：模仿目标的行为、兴趣
- 互惠原则：先给予后索取
- 社会证明：利用群体压力

### 2. 信息收集
```
公开信息 → 个人画像 → 弱点识别 → 攻击设计
```

**信息来源：**
- 社交媒体 → [[社交媒体信息收集]]
- 公司网站/新闻
- 公共记录
- 丢弃的文件

### 3. 操纵技巧
```
触发情感 → 降低判断力 → 引导行为
```

**常见技巧：**
- 紧迫感："立即行动，否则..."
- 恐惧："你的账户被盗"
- 好奇心："看看这个秘密"
- 贪婪："你中奖了"

## 认知偏差利用

| 偏差类型 | 说明 | 社会工程学应用 |
|---------|------|---------------|
| **权威偏差** | 倾向服从权威 | 伪装高管、IT 部门 |
| **互惠原则** | 感觉有义务回报 | 先帮忙再索取 |
| **稀缺效应** | 稀有物品更有价值 | "仅限今天" |
| **社会认同** | 跟随他人行为 | "大家都已安装" |
| **承诺一致性** | 保持与承诺一致 | 先小承诺后大要求 |
| **友好偏差** | 信任喜欢的人 | 建立良好关系 |

## 攻击流程模型

### Kill Chain 模型

```
1. 调查 (Reconnaissance)
   ↓ 信息收集 → [[社交媒体信息收集]]
2. 诱饵 (Hook)
   ↓ 设计诱因
3. 戏剧 (Play)
   ↓ 执行操纵
4. 退出 (Exit)
   ↓ 撤退/转移
5. 洗钱/利用 (Follow-up)
```

---

## 与技术攻击的结合

社会工程学常作为技术攻击的"敲门砖"：

| 社会工程学 | 技术攻击 | 结合案例 |
|-----------|---------|---------|
| 钓鱼邮件 | 恶意软件 | 邮件携带恶意附件 |
| 伪造身份 | 内网渗透 | 假装 IT 远程维护 |
| 信息收集 | 密码破解 | 用个人信息猜测密码 |
| 伪装 | 物理入侵 | 假装快递员进入 |

## 防御原则

### 个人层面
- **质疑一切**：验证身份，不过度信任
- **保护信息**：谨慎分享个人信息
- **多因素验证**：启用 [[访问控制]] 中的 MFA
- **培训意识**：了解常见手法

### 组织层面
- **流程化验证**：敏感操作多重确认
- **最小权限**：限制信息访问范围
- **定期培训**：模拟钓鱼演练
- **文化建设**："报告可疑"而非"害怕犯错"

## 伦理考量

| 合法用途 | 非法用途 |
|---------|---------|
| 授权渗透测试 | 未授权获取信息 |
| 安全意识培训 | 欺诈/诈骗 |
| 红队演练 | 身份盗窃 |
| 安全研究 | 勒索/敲诈 |

**关键区分：授权与意图**
