# 零信任架构

## 核心理念

**"永远不信任，始终验证"**

零信任假设网络内部和外部一样危险，不再依赖传统的边界防护（防火墙、VPN），而是对每一次访问请求都进行身份验证和授权。

---

## 传统架构 vs 零信任

| 传统架构 | 零信任架构 |
|---------|-----------|
| 信任内网，不信任外网 | 不信任任何网络 |
| 边界防护（防火墙） | 身份是新的边界 |
| 一次验证，持续信任 | 持续验证，动态授权 |
| 网络位置决定权限 | 身份和上下文决定权限 |

---

## 零信任三大原则

### 1. 显式验证
始终基于所有可用的数据点进行身份验证和授权：
- 身份
- 位置
- 设备健康状态
- 服务/工作负载
- 数据分类

### 2. 最小权限访问
- 仅授予完成任务所需的最小权限
- Just-in-time（即时）访问
- Just-enough（够用）权限

### 3. 假设已被攻破
- 最小化爆炸半径
- 端到端加密
- 微分段
- [[审计日志]] 与监控

---

## 架构组件

```
┌─────────────────────────────────────────────────┐
│                   策略决策点 (PDP)                │
│  - 身份验证  - 设备评估  - 风险判断  - 策略执行  │
└─────────────────────────────────────────────────┘
                      ↕
┌─────────────────────────────────────────────────┐
│                   策略执行点 (PEP)                │
│  - API 网关  - 代理  - 防火墙  - 应用网关       │
└─────────────────────────────────────────────────┘
```

**核心组件：**
- 身份提供商 (IdP)
- 设备管理系统 (MDM)
- 策略引擎
- [[访问控制]] 系统
- [[审计日志]] 系统

---

## 实施路线图

| 阶段 | 内容 |
|------|------|
| **可视** | 发现资产、梳理数据流、识别访问模式 |
| **微分段** | 网络分段、应用隔离、最小化信任域 |
| **身份化** | 统一身份认证、多因素认证 (MFA) |
| **策略化** | 基于风险的动态访问策略 |
| **持续验证** | 持续监控、自动响应 |

---

## 适用场景

- 云原生应用
- 远程办公/混合办公
- 第三方访问管理
- 高安全要求行业（金融、医疗、政府）
