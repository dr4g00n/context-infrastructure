# 威胁情报核心理念

## 核心原则

威胁情报不是简单的"漏洞数据库"，而是关于**对手、能力和意图**的知识体系，用于理解威胁环境、预测攻击行为、指导防御决策。

> "知己知彼，百战不殆" — 《孙子兵法》

在网络安全领域，**威胁情报 = 对手的画像 + 攻击的模式 + 防御的知识**

---

## 1. 情报的定义

### 数据 vs 信息 vs 情报

| 层次 | 定义 | 价值 | 示例 |
|------|------|------|------|
| **数据** | 原始的事实和数字 | 低 | IP 地址 192.168.1.1 |
| **信息** | 有组织的数据 | 中 | IP 192.168.1.1 与已知 C2 服务器通信 |
| **情报** | 可行动的知识 | 高 | APT 组织正在针对金融行业进行钓鱼攻击 |

### 情报的价值链条

```
数据收集 → 信息处理 → 情报分析 → 决策支持
    ↓          ↓          ↓          ↓
  原始数据   关联分析   风险评估   行动建议
```

---

## 2. 情报分级

### 战略情报（Strategic Intelligence）

**目标受众**：高管、决策者

**内容**：
- 行业威胁趋势
- 地缘政治影响
- 长期威胁预测
- 风险投资决策

**时间范围**：6-24 个月

**示例**：
- "未来一年金融行业将面临 40% 的 APT 攻击增长"
- "地缘冲突将导致关键基础设施攻击增加"

### 战术情报（Tactical Intelligence）

**目标受众**：安全运营、防御团队

**内容**：
- 攻击者 TTPs（战术、技术、流程）
- IOC（入侵指标）
- 漏洞利用信息
- 检测规则

**时间范围**：数小时到数周

**示例**：
- "攻击者使用 PowerShell 无文件攻击技术"
- "检测到新的 C2 域名：malicious.example.com"

### 操作情报（Operational Intelligence）

**目标受众**：事件响应、红队

**内容**：
- 具体攻击活动
- 单次事件详情
- 实时威胁信息

**时间范围**：实时到数小时

**示例**：
- "正在进行的钓鱼攻击活动"
- "检测到新的勒索软件变种"

## 3. 情报生命周期

### 1. 方向与规划

明确情报需求：
- 需要了解哪些威胁？
- 情报将如何被使用？
- 优先级是什么？

### 2. 收集

**数据来源**：

| 来源类型 | 示例 | 价值 |
|---------|------|------|
| **开源情报（OSINT）** | 威胁报告、博客、论坛 | 高覆盖、低成本 |
| **商业情报** | 商业威胁情报平台 | 高质量、结构化 |
| **政府情报** | CISA、NSA 报告 | 权威、国家级 |
| **行业共享** | ISAC、信息共享组织 | 行业相关 |
| **内部数据** | 日志、事件 | 最相关、定制化 |

### 3. 处理

- 数据清洗
- 格式标准化
- 去重与关联
- 数据 enrichment

### 4. 分析

**分析方法**：

| 方法 | 目的 | 输出 |
|------|------|------|
| **关联分析** | 发现关联关系 | 攻击者画像、攻击链 |
| **趋势分析** | 识别威胁趋势 | 威胁预测 |
| **行为分析** | 理解攻击行为 | TTPs 提取 |
| **归因分析** | 识别攻击者 | APT 组织归属 |

### 5. 传播

**传播形式**：
- 情报报告
- 检测规则（Sigma、YARA、Snort）
- IOC 馈送
- 仪表盘和可视化

### 6. 反馈

- 情报有效性评估
- 误报率分析
- 防御效果验证
- 持续改进

## 4. 关键概念

### IOC（入侵指标）

可检测的攻击痕迹：

| IOC 类型 | 示例 | 检测方式 |
|---------|------|---------|
| 网络IOC | IP、域名、URL | 网络流量监控 |
| 文件IOC | 哈希、文件名 | 端点检测 |
| 邮件IOC | 发件人、主题 | 邮件网关 |
| 行为IOC | 进程名、注册表 | EDR |

**局限性**：
- 易被攻击者绕过（改变 IOC）
- 事后性（需要已知攻击）
- 大量误报

### TTPs（战术、技术、流程）

攻击者的行为模式：

```
战术（Tactic）→ "为什么"
    │
    └── "获取凭据"

技术（Technique）→ "怎么做"
    │
    └── "钓鱼攻击"（T1566）

流程（Procedure）→ "具体细节"
    │
    └── "使用带恶意附件的钓鱼邮件"
```

**MITRE ATT&CK 框架**：
- 战术：12 个战术目标
- 技术：数百种具体技术
- 缓解措施：对应的防御方法

### 威胁猎杀（Threat Hunting）

主动寻找威胁而非被动等待：

```
假设驱动
    ↓
    "攻击者可能使用 PowerShell 进行横向移动"
    ↓
设计查询
    ↓
    "搜索远程 PowerShell 调用"
    ↓
数据分析
    ↓
    发现异常行为
    ↓
响应与改进
```

## 5. 情报标准与框架

### STIX（结构化威胁信息表达）

威胁信息的标准化格式：

```json
{
  "type": "indicator",
  "id": "indicator--uuid",
  "pattern": "[domain-name:value = 'malicious.com']",
  "valid_from": "2024-01-01T00:00:00.000Z"
}
```

### TAXII（可信自动化情报交换）

情报信息的自动共享协议

### MITRE ATT&CK

对手战术和技术知识库：
- Enterprise（企业）
- Mobile（移动）
- ICS（工控）

### CVE / CVSS

- CVE：通用漏洞披露
- CVSS：通用漏洞评分系统

## 6. 情报驱动的防御

### 从被动到主动

```
传统防御（被动）              情报驱动防御（主动）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
等待攻击发生          →      预测攻击可能性
基于已知签名          →      理解对手行为
事后响应              →      提前部署防御
通用防护              →      针对性防御
```

### 情报应用场景

| 场景 | 情报应用 | 效果 |
|------|---------|------|
| **漏洞优先级** | 利用情报判断哪些漏洞被真实利用 | 聚焦关键风险 |
| **事件响应** | 快速识别攻击者身份和意图 | 加速响应 |
| **威胁猎杀** | 基于情报设计猎杀假设 | 主动发现 |
| **安全架构** | 了解对手能力调整防御 | 对抗性设计 |
| **红队演练** | 模拟真实攻击者行为 | 提升演练质量 |

## 7. 情报质量评估

### 评估维度

| 维度 | 问题 |
|------|------|
| **相关性** | 情报与我们的环境相关吗？ |
| **时效性** | 情报是最新的吗？ |
| **准确性** | 情报来源可靠吗？ |
| **可操作性** | 情报能指导行动吗？ |
| **完整性** | 情报提供足够上下文吗？ |

### 误报管理

- 建立反馈机制
- 定期清理无效 IOC
- 优化检测规则阈值
- 使用上下文减少误报

---

## 8. 常见挑战

### 挑战与应对

| 挑战 | 应对策略 |
|------|---------|
| **信息过载** | 建立情报优先级，聚焦相关威胁 |
| **误报率高** | 上下文 enrichment，多源验证 |
| **缺乏行动能力** | 确保情报与防御能力匹配 |
| **共享困难** | 使用标准格式（STIX） |
| **成本高昂** | 开源与商业情报结合 |
| **人才短缺** | 培训现有团队，使用托管服务 |

## 9. 与其他领域的关系

### 与渗透测试

```
渗透测试 ← 威胁情报
    ↓           ↓
验证防御   ←   了解真实对手
    ↓           ↓
发现漏洞   ←   模拟真实攻击
    ↓           ↓
改进防御   ←   对抗性设计
```

**应用**：
- 基于 ATT&CK 设计测试用例
- 模拟特定 APT 组织的攻击
- 验证对已知威胁的防御能力

### 与社会工程

- OSINT 是威胁情报的重要组成部分
- 钓鱼攻击情报指导防御策略
- 人员情报支持内部威胁评估

### 与纵深防御

- 威胁情报指导防御优先级
- 情报驱动的检测规则
- 对手知识支持防御策略调整
