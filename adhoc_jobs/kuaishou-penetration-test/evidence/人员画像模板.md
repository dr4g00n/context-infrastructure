# 人员画像模板

## 模板说明

本模板用于渗透测试中人员情报收集和分析，为后续社会工程学测试提供目标画像。

---

## 基础信息模板

### 字段说明

| 字段 | 说明 | 来源 |
|------|------|------|
| **姓名** | 真实姓名或化名 | 公开资料 |
| **性别** | 男/女 | 公开资料 |
| **年龄** | 或年龄段 | 公开资料/推测 |
| **职位** | 当前职位 | LinkedIn、脉脉 |
| **部门** | 所属部门 | 公开资料 |
| **级别** | 职级（P/M序列） | 招聘JD、脉脉 |
| **入职时间** | 或入职年限 | LinkedIn、脉脉 |
| **办公地点** | 工作城市 | 公开资料 |
| **教育背景** | 学历、学校 | LinkedIn、脉脉 |

### 模板代码

```yaml
# 人员基础信息
basic_info:
  name: ""
  gender: ""
  age: ""
  age_range: ""
  position: ""
  department: ""
  level: ""
  tenure: ""
  location: ""
  education:
    - degree: ""
      school: ""
      major: ""
      year: ""
```

## 社会工程价值评估

### 评估维度

| 维度 | 说明 | 评分标准 |
|------|------|---------|
| **信息价值** | 掌握信息的价值程度 | 1-5 分 |
| **系统权限** | 可接触系统的权限级别 | 1-5 分 |
| **钓鱼易感性** | 成为钓鱼目标的难易 | 1-5 分 |
| **影响力** | 在组织内的影响力 | 1-5 分 |
| **可达性** | 接触的难易程度 | 1-5 分 |

### 模板代码

```yaml
# 社会工程价值评估
se_value_assessment:
  information_value: 1  # 1-5
  system_access: 1      # 1-5
  phishing_susceptibility: 1  # 1-5
  influence: 1          # 1-5
  accessibility: 1      # 1-5

  overall_score: 1      # 总分 5-25

  suitability:
    phishing_target: false
    information_source: false
    technical_understanding: false
    social_engineering_target: false

  attack_vectors: []    # 可能的攻击方式
  mitigation: []        # 防御建议
```

## 完整示例

### 示例：技术总监画像

```markdown
# 人员画像 - 示例

## 基础信息
- **姓名**：张三（化名）
- **性别**：男
- **年龄**：35-40 岁
- **职位**：技术总监
- **部门**：技术部
- **级别**：P9+
- **办公地点**：北京

## 职业背景
### 工作经历
- 2018-至今：快手，技术总监
- 2015-2018：某互联网大厂，架构师
- 2012-2015：某创业公司，技术负责人

### 专业技能
- 技术栈：Java、Go、微服务、分布式系统
- 架构能力：大规模系统设计、高可用架构
- 管理能力：百人团队管理、技术规划

## 技术画像
### 技术关注
- 微服务架构演进
- 云原生技术
- DevSecOps

### 技术观点
- "系统稳定性优于新功能"
- "安全应该是架构设计的核心考虑"

## 社会工程价值评估
| 维度 | 评分 | 说明 |
|------|------|------|
| 信息价值 | 5/5 | 掌握核心架构信息 |
| 系统权限 | 5/5 | 高权限访问 |
| 钓鱼易感性 | 2/5 | 技术背景，警惕性高 |
| 影响力 | 5/5 | 技术决策者 |
| 可达性 | 2/5 | 高层，难以接触 |

**总分**：19/25

### 适用场景
- [x] 技术了解目标
- [x] 架构信息收集
- [ ] 钓鱼攻击目标（不推荐）
- [ ] 社会工程目标（不推荐）
```

## 输出格式

### 文件命名规范

```
personas/
├── 01-{姓名/角色}-基础画像.md
├── 02-{姓名/角色}-技术画像.md
├── 03-{姓名/角色}-社交画像.md
└── summary-人员画像汇总.md
```

### 报告格式

```markdown
# {项目名称} 人员画像报告

## 1. 画像概览
[人员清单、分类统计]

## 2. 高价值目标
[根据社会工程价值评估排序]

## 3. 技术团队画像
[技术团队成员画像]

## 4. 安全团队画像
[安全团队成员画像]

## 5. 管理层画像
[高管团队成员画像]

## 6. 社会工程建议
[基于画像的攻击场景建议]

## 7. 风险提示
[使用注意事项]
```
