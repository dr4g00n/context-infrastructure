# 供应商信息管理

**目录用途**：统一管理电子吧唧项目所有供应商信息

---

## 目录结构

```
suppliers/
├── README.md                   # 本文件，使用说明
├── 供应商信息模板.md            # 供应商信息记录模板
├── SUPP-001-金芯智造.md        # 供应商档案（示例）
├── SUPP-002-供应商名.md        # 供应商档案
└── ...
```

---

## 使用指南

### 1. 新建供应商档案

1. 复制 `供应商信息模板.md`
2. 重命名为 `SUPP-{编号}-{公司简称}.md`
3. 填写供应商信息

### 2. 供应商编号规则

- 格式：`SUPP-{三位数字}`
- 示例：`SUPP-001`、`SUPP-002`...
- 建议：按接触顺序编号

### 3. 文件命名规范

```
SUPP-{编号}-{公司简称}.md
```

示例：
- `SUPP-001-金芯智造.md`
- `SUPP-002-色克龙制造.md`
- `SUPP-003-创意潮玩.md`

## 模板章节说明

| 章节 | 说明 | 必填 |
|------|------|------|
| 1. 基本信息 | 公司基本信息、联系方式 | ✅ |

| 3. 商务条件 | 价格、MOQ、交期 | ✅ |
| 4. 资质认证 | ISO、CE等认证 | 🟡 |

| 7. 样品记录 | 样品采购和评估 | 🟡 |
| 8. 合作历史 | 订单记录、质量记录 | 🟡 |
| 9. 决策建议 | 合作定位、下一步 | ✅ |

*✅ 必填 | 🟡 建议填写*

## 信息更新流程

### 初次接触

```
初步联系 → 创建档案 → 填写基本信息 → 记录沟通内容
```

### 深度沟通

```
获取报价 → 更新商务条件 → 获取规格书 → 更新产品信息
```

### 样品评估

```
采购样品 → 收货测试 → 填写评估报告 → 更新决策建议
```

### 持续维护

```
每次沟通 → 更新沟通记录 → 定期评估 → 调整合作定位
```

## 供应商分类

### 按合作状态

| 状态 | 标签 | 说明 |
|------|------|------|
| 🔴 主供应商 | `primary` | 核心合作伙伴 |
| 🟡 备用供应商 | `backup` | 风险分散 |
| 🟢 候选供应商 | `candidate` | 潜在合作对象 |
| ⚪ 观察对象 | `watch` | 暂不考虑 |
| ⚫ 已淘汰 | `archived` | 不再考虑 |

| LED胸牌类 | `led-badge` |
| 智能徽章类 | `smart-badge` |
| 其他 | `other` |

## 快速检索

### 查看所有供应商

```bash
ls suppliers/*.md | grep SUPP
```

### 按状态筛选

```bash
# 查看主供应商
grep -l "🔴 \*\*主供应商\*\*" suppliers/*.md

# 查看候选供应商
grep -l "🟢 \*\*候选供应商\*\*" suppliers/*.md
```

### 搜索供应商

```bash
# 按公司名搜索
grep -r "公司全称" suppliers/
```
